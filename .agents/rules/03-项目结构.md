# 项目结构

## 基本原则（NON-NEGOTIABLE）

本项目目录结构已确定，AI 必须遵循现有架构。

必须：

- 优先复用已有目录，不得创建重复职责目录。
- 保持目录职责单一，一个目录只负责一类资源。
- 新增代码必须放入对应职责目录。
- 保持与当前项目结构一致。

禁止：

- 创建 `api`、`services`、`hooks`、`pages` 等与现有目录职责重复的目录。
- 修改已有目录结构。
- 将不同职责的文件混放。

---

# 项目目录

```
src
├── assets          # 静态资源
├── components      # 公共组件
├── composables     # 可复用组合式函数
├── http            # Axios 请求封装
├── layouts         # 页面布局
├── router          # Vue Router
├── stores          # Pinia
├── styles          # 全局样式
├── types           # 全局类型
├── utils           # 工具函数
├── views           # 页面
├── App.vue
└── main.ts
```

AI 必须保持以上目录结构，不得随意新增同职责目录。

---

# assets

目录：

```text
src/assets
```

用于存放：

- 图片
- SVG
- Icon
- 字体
- 视频
- 其它静态资源

禁止存放：

- Vue 组件
- TypeScript 文件
- HTTP 请求
- Store

---

# components

目录：

```text
src/components
```

用于存放：

可复用公共组件。

例如：

```text
BaseTable.vue
BaseDialog.vue
SvgIcon.vue
Pagination.vue
```

禁止：

业务页面组件。

例如：

```text
UserTable.vue
SearchForm.vue
EditDialog.vue
```

业务组件应放在：

```text
src/views/**/components
```

---

# composables

目录：

```text
src/composables
```

用于存放：

可复用 Composition API 逻辑。

统一命名：

```text
useXxx.ts
```

例如：

```text
useRequest.ts
usePagination.ts
useTable.ts
```

禁止：

存放页面专属业务逻辑。

---

# http

目录：

```text
src/http
```

统一管理所有 Axios 请求。

要求：

- 一个业务模块一个文件。
- 不在 Vue 组件中直接使用 Axios。
- 请求参数、响应类型遵循项目 API 规范。

例如：

```text
http
├── user.ts
├── login.ts
├── experiment.ts
```

新增接口请遵循：

```
.agents/skills/create-api
```

---

# layouts

目录：

```text
src/layouts
```

用于存放页面布局。

例如：

```text
DefaultLayout.vue
BlankLayout.vue
```

禁止：

业务页面放入 layouts。

---

# router

目录：

```text
src/router
```

统一维护：

- 路由配置
- 路由守卫
- 动态路由

禁止：

页面内部维护路由配置。

新增路由请遵循：

```
.agents/skills/create-route
```

---

# stores

目录：

```text
src/stores
```

统一使用：

Pinia。

要求：

- 一个业务模块一个 Store。
- Store 只负责共享状态。
- 页面私有状态不要放入 Store。

例如：

```text
stores
├── app.ts
├── user.ts
├── permission.ts
```

新增 Store 请遵循：

```
.agents/skills/create-store
```

---

# styles

目录：

```text
src/styles
```

统一维护：

- 全局样式入口
- Reset
- Variables
- Mixins
- Theme
- Animation

推荐结构：

```text
styles
├── index.scss
├── reset.scss
├── variables.scss
├── mixins.scss
├── theme.scss
└── animation.scss
```

要求：

- `main.ts` 仅引入 `styles/index.scss`
- 全局样式统一由 `index.scss` 聚合
- 页面样式写在对应 `.vue` 文件中
- 禁止多个页面重复引入公共样式

---

目录：

```text
src/types
```

用于存放：

全局 TypeScript 类型。

例如：

```text
global.ts
env.ts
common.ts
```

例如：

```text
views/User/types.ts
```

# utils

目录：

```text
src/utils
```

用于存放：

公共工具函数。

例如：

```text
date.ts
storage.ts
download.ts
validate.ts
```

禁止：

将业务逻辑放入 utils。

---

# views

目录：

```text
src/views
```

每个业务模块一个目录。

推荐结构：

```text
views
├── Login
├── Home
├── User
└── Experiment
```

每个业务目录推荐：

```text
User
├── index.vue
├── components
├── composables
├── constants.ts
├── types.ts
└── assets
```

要求：

- 一个业务一个目录。
- 页面入口统一使用 `index.vue`。
- 页面私有组件放在 `components`。
- 页面私有组合函数放在 `composables`。

禁止：

所有页面直接放在 `views` 根目录。

---

# App.vue

文件：

```text
src/App.vue
```

职责：

- 应用根组件。
- 提供 Layout 容器。
- 提供 RouterView。
- 不编写业务逻辑。

---

# main.ts

文件：

```text
src/main.ts
```

职责：

- 创建 Vue 应用。
- 注册 Pinia。
- 注册 Vue Router。
- 注册 Element Plus。
- 引入全局样式。
- 挂载 App。

禁止：

- 编写业务逻辑。
- 编写页面逻辑。
- 编写接口请求。

---

# AI 开发约定

新增代码前必须：

1. 检查对应目录是否已存在。
2. 优先复用已有目录。
3. 不创建重复职责目录。
4. 保持目录职责单一。
5. 保持与项目现有架构一致。

新增目录前必须确认：

- 当前项目是否已有相同职责目录。
- 是否确实存在新增目录的必要。
