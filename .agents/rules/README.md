# 使用说明

开发过程中，根据需要查阅对应的 Rules 文件即可。

例如：

| 需求                 | 对应 Rules   |
| -------------------- | ------------ |
| 了解项目整体架构     | 01-项目概述  |
| 编写 TypeScript 代码 | 02-编码规范  |
| 新增页面             | 03-项目结构  |
| 编写 Vue 组件        | 04-组件规范  |
| 调用后端接口         | 05-API规范   |
| 配置路由             | 06-路由规范  |
| 使用 Pinia           | 07-Pinia规范 |
| 查看通用开发约束     | 08-通用约束  |
| 编写样式             | 09-样式规范  |
| 编写文档             | 10-文档规范  |
| 提交前检查           | 11-测试规范  |

# 维护原则

新增规范时：

1. 按顺序新增编号文件（如 `12-xxx规范.md`）。
2. 同步更新本 README。
3. 保持每个规范职责单一，避免重复内容。
4. 若涉及多个模块，可在对应 Rules 中相互引用。
