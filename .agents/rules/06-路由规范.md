# 路由规范

## 基本原则（NON-NEGOTIABLE）

本项目统一使用 Vue Router 4。

必须：

- 所有路由统一维护在 `src/router` 下。
- 路由统一由 `src/router/index.ts` 创建和导出。
- 页面组件统一放在 `src/views` 下。
- 每个页面对应一个路由。
- 页面组件统一使用懒加载（首页等特殊页面除外）。
- 页面入口统一命名为 `index.vue`。

禁止：

- 在组件中创建 Router 实例。
- 在组件中维护路由配置。
- 使用 `window.location.href` 进行页面跳转。
- 创建重复的路由。

---

# Router

Router 实例统一维护：

```
src/router/index.ts
```

禁止：

- 创建多个 Router。
- 在其它目录维护路由。

---

# 页面目录

页面统一放在：

```
src/views
```

推荐结构：

```
views
├── Home
│   └── index.vue
├── User
│   └── index.vue
└── Experiment
    └── index.vue
```

要求：

- 一个页面一个目录。
- 页面入口统一使用 `index.vue`。

---

# 路由配置

新增路由统一在：

```
src/router/index.ts
```

维护。

推荐：

```ts
{
  path: '/user',
  name: 'User',
  component: () => import('@/views/User/index.vue'),
  meta: {
    title: '用户管理'
  }
}
```

要求：

- 使用懒加载。
- `name` 与页面目录保持一致。
- `meta` 至少包含 `title`。

---

# 路由命名

Path：

```
/user
```

Name：

```
User
```

要求：

- Name 使用 PascalCase。
- Path 使用 kebab-case。
- Name 与目录名称保持一致。

例如：

```
views
└── UserCenter
```

对应：

```ts
{
  path: '/user-center',
  name: 'UserCenter'
}
```

---

# 页面跳转

统一使用：

```ts
const router = useRouter();
```

页面参数：

```ts
const route = useRoute();
```

页面跳转：

```ts
router.push('/user');
```

或：

```ts
router.push({
    name: 'User'
});
```

禁止：

```ts
window.location.href = '/user';
```

---

# 路由参数

Query：

```
/user?page=1
```

使用：

```ts
route.query
```

Params：

```
/user/:id
```

使用：

```ts
route.params
```

禁止手动解析 URL。

---

# Meta

推荐：

```ts
meta: {
    title: '用户管理'
}
```

以后新增：

```
requiresAuth
keepAlive
permission
```

必须保持统一。

---

# AI 开发约定

新增页面时：

必须：

1. 创建页面目录。
2. 创建 `index.vue`。
3. 注册路由。
4. 使用懒加载。
5. 设置 `meta.title`。

不得：

- 创建重复路由。
- 创建未注册页面。
- 修改已有路由行为。
