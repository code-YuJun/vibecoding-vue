# 样式规范

## 基本原则（NON-NEGOTIABLE）

项目统一采用：

- Vue Single File Component（SFC）
- `<style scoped lang="scss">`
- SCSS 编写样式
- Element Plus 作为基础组件库

禁止：

- 使用内联样式（style 属性）
- 在组件中大量使用 style 动态拼接
- 使用全局 CSS 覆盖业务样式
- 编写无法复用的大量重复样式

---

# Style 标签规范

统一：

```vue
<style scoped lang="scss">

</style>
```

禁止：

```vue
<style>

</style>
```

禁止：

```vue
<style lang="css">

</style>
```

---

# CSS 类命名

推荐：

```scss
.page {}

.header {}

.content {}

.footer {}

.table {}

.form {}

.card {}

.empty {}

.loading {}
```

组件内部推荐：

```scss
.container {}

.title {}

.subtitle {}

.actions {}

.list {}

.item {}
```

禁止：

```scss
.box1 {}

.div {}

.left {}

.right {}

.red {}
```

命名应体现业务含义。

---

# SCSS 编写规范

推荐：

```scss
.container {
  display: flex;

  .title {
    font-size: 16px;
  }

  .content {
    padding: 16px;
  }
}
```

禁止：

过深嵌套：

```scss
.a {
  .b {
    .c {
      .d {
        .e {}
      }
    }
  }
}
```

建议：

嵌套层级不超过三层。

---

# 布局规范

优先使用：

- Flex
- Grid（复杂布局）

例如：

```scss
display: flex;
```

避免：

```scss
float
```

除兼容特殊场景外，不使用 Float 布局。

# 颜色规范（NON-NEGOTIABLE）

优先使用：

- Element Plus Theme Token
- CSS Variables

禁止：

```scss
color: #409EFF;

background: #67C23A;
```

推荐：

```scss
color: var(--el-color-primary);

background: var(--el-bg-color);
```

如果项目新增自定义主题变量：

统一维护：

```
src/styles/variables.scss
```

禁止组件自行定义主题颜色。

# Element Plus 使用规范

优先使用：

- el-button
- el-input
- el-select
- el-form
- el-table
- el-dialog

不要自行重新实现基础组件。

如需扩展：

采用二次封装。

例如：

```
components

BaseTable

BaseDialog

BaseForm
```

---

# Scoped 样式

组件默认：

```vue
<style scoped lang="scss">
```

如需修改 Element Plus 内部样式：

统一：

```scss
:deep(.el-table) {

}
```

禁止：

直接删除 scoped。

---

# Figma 设计稿还原

当已连接 Figma MCP 时：

优先读取设计稿真实信息：

- 字体大小
- 字体粗细
- 行高
- 间距
- 圆角
- 阴影
- 模糊
- 边框
- 颜色

禁止：

凭经验猜测。

---

# Figma 颜色转换

例如：

```
#13131C73
```

转换：

```scss
background: rgba(19, 19, 28, 0.45);
```

不要保留：

```
#13131C73
```

---

# 图片与图标

图片：

统一放：

```
src/assets/images
```

SVG：

统一放：

```
src/assets/icons
```

禁止：

引用网络占位图片。

禁止：

临时复制 SVG 到组件内部。

---

# 占位资源

设计资源未提供时：

允许：

```html
<div class="placeholder-image"></div>

<div class="placeholder-icon"></div>
```

必须：

```ts
// TODO: 等待设计资源
```

---

# 动画

推荐：

```
transition
```

简单交互。

复杂动画：

使用：

```
animation
```

避免：

大量 JS 操作 DOM 实现动画。

---

# AI 开发约定

AI 生成页面时：

必须：

- 优先使用 Element Plus。
- 优先使用已有组件。
- 保持与现有页面风格一致。
- 使用 scoped SCSS。
- 使用 CSS Variables。
- 不新增无意义样式。
