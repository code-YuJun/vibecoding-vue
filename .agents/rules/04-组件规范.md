# 组件规范

## 基本原则（NON-NEGOTIABLE）

所有 Vue 组件必须遵循：

- 单一职责
- 高内聚、低耦合
- 保持组件可复用
- 保持类型完整
- 保持结构清晰
- 优先复用已有组件

禁止：

- 超大组件
- 页面业务全部写在一个组件中
- 重复封装已有能力

---

# 组件编写方式（NON-NEGOTIABLE）

所有组件统一使用：

```vue
<script setup lang="ts">
```

禁止：

- Options API
- JavaScript
- Class Component

推荐结构：

```vue
<script setup lang="ts">
// import

// props

// emits

// composables

// state

// computed

// watch

// methods

// lifecycle
</script>

<template>

</template>

<style scoped lang="scss">

</style>
```

---

# 组件命名

组件文件统一采用：

```
PascalCase
```

例如：

```
UserTable.vue

SearchForm.vue

CreateDialog.vue

UploadImage.vue
```

禁止：

```
user.vue

table.vue

index.vue（业务组件）
```

> `index.vue` 仅允许作为页面入口文件使用。

---

# Props

Props 必须声明类型。

例如：

```ts
interface Props {
  loading?: boolean;
  data: User[];
}

const props = defineProps<Props>();
```

禁止：

```ts
const props = defineProps();
```

禁止：

```ts
type: Object

type: Array
```

优先使用 TypeScript 类型。

---

# Emits

所有组件事件必须声明。

例如：

```ts
const emit = defineEmits<{
  submit: [];
  cancel: [];
  change: [value: string];
}>();
```

禁止：

```ts
emit("submit");
```

但未声明事件。

---

# v-model

推荐使用：

```ts
defineModel()
```

（若项目 Vue 版本支持）

否则：

使用：

```
modelValue

update:modelValue
```

保持 Vue 官方规范。

---

# Ref

推荐：

```ts
const tableRef = ref();
```

命名：

```
xxxRef
```

例如：

```ts
formRef

tableRef

dialogRef
```

---

# Computed

派生数据统一使用：

```ts
computed()
```

禁止：

在模板中编写复杂表达式。

例如：

避免：

```vue
{{ list.filter(...).map(...) }}
```

应：

```ts
const tableData = computed(...)
```

---

# Watch

仅监听真正需要监听的数据。

推荐：

```ts
watch(
  () => props.id,
  loadData
);
```

避免：

```ts
watch(
    props,
    ...
)
```

深度监听整个对象。

---

# 生命周期

统一使用 Composition API 生命周期。

例如：

```ts
onMounted()

onUnmounted()

onActivated()

onDeactivated()
```

禁止：

Options API 生命周期。

---

# 组件通信

父子组件：

使用：

```
Props

Emits
```

跨层级：

根据项目情况：

- Pinia
- provide/inject（仅限局部）

禁止：

- EventBus
- window 全局变量

---

# Element Plus

项目统一使用：

```
Element Plus
```

要求：

- 优先使用官方组件
- 优先使用官方图标
- 保持官方交互方式
- 保持统一风格

如果项目已有公共组件：

优先复用。

禁止重复封装：

- Button
- Table
- Dialog

等已有能力。

---

# 页面组件

页面组件仅负责：

- 页面布局
- 页面数据组织
- 页面业务流程

复杂业务应拆分：

例如：

```
pages/
└── User/
    ├── index.vue
    ├── components/
    │   ├── SearchForm.vue
    │   ├── UserTable.vue
    │   └── EditDialog.vue
```

---

# 公共组件

公共组件放置：

```
src/components
```

要求：

- 与业务无关
- 可复用
- 类型完整
- 文档清晰

业务组件：

放置：

```
当前模块/components
```

禁止放入：

```
src/components
```

---

# Composable

可复用逻辑统一抽离：

```
src/composables
```

命名：

```
useXxx
```

例如：

```
useTable

usePagination

useUser

useRequest
```

禁止：

将大量业务逻辑直接写入组件。

---

# 样式

统一：

```scss
<style scoped lang="scss">
```

推荐：

- scoped
- BEM
- CSS Variables（如项目已使用）

禁止：

- 行内 style
- 大量全局样式
- !important 滥用

---

# HTTP 请求

组件内禁止直接编写 Axios 请求。

统一调用：

```
src/http
```

例如：

```ts
import { getUserList } from "@/http/user";
```

如需新增接口：

遵循：

```
.agents/skills/create-api
```

---

# Store 使用

组件内：

优先复用已有 Store。

例如：

```ts
const userStore = useUserStore();
```

禁止：

重复维护相同状态。

如需新增 Store：

遵循：

```
.agents/skills/create-store
```

---

# AI 开发约定

AI 创建组件时必须：

- 优先检查是否已有相似组件。
- 优先复用公共组件。
- 保持 Props、Emits 类型完整。
- 保持组件职责单一。
- 保持组件体积适中。
- 保持最小修改范围。
- 不重复实现已有能力。
- 与项目现有代码风格保持一致。
