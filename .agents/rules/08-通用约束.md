# 通用约束

## AI 开发原则（NON-NEGOTIABLE）

AI 在新增或修改代码前，必须遵循以下原则：

1. 优先复用已有代码，不重复实现已有能力。
2. 优先修改已有文件，不随意创建新文件。
3. 优先遵循当前项目架构，不擅自调整目录结构。
4. 保持改动最小化，仅修改与当前需求相关的内容。
5. 不删除已有代码，除非用户明确要求。

禁止：

- 创建重复组件。
- 创建重复 Store。
- 创建重复 API。
- 创建重复工具函数。
- 创建重复 Composable。

---

# 搜索优先原则（NON-NEGOTIABLE）

新增代码前必须检查项目是否已有相同能力。

检查顺序：

1. Components
2. Composables
3. Stores
4. Http
5. Utils
6. Types

若已有实现：

- 优先复用。
- 必要时扩展已有实现。

禁止重新创建同职责模块。

# 中文规范（NON-NEGOTIABLE）

所有文档、注释统一使用中文。

包括：

- README
- OpenSpec
- Skills
- Rules
- 代码注释

代码中的：

- 变量
- 函数
- 类
- 类型
- 文件名

统一使用英文。

---

# 路径规范

项目统一使用路径别名：

```ts
@
```

例如：

```ts
import UserTable from '@/components/UserTable.vue';
```

禁止：

```ts
import UserTable from '../../../components/UserTable.vue';
```

---

# Auto Import

本项目已配置：

- Auto Import
- Components Auto Import

对于已自动导入的 API：

禁止重复编写 import。

例如：

```ts
ref
computed
watch
onMounted
defineStore
useRouter
useRoute
```

应直接使用。

---

# 环境变量

禁止硬编码：

- 接口地址
- 域名
- Token
- 环境配置

统一使用：

```
.env
.env.development
.env.production
```

读取：

```ts
import.meta.env
```

---

# 占位资源规范

以下情况允许使用占位元素：

- 图标资源未提供
- 图片资源未提供
- 第三方资源暂未接入

推荐：

```vue
<div class="placeholder-image" />

<div class="placeholder-icon" />
```

必须添加：

```ts
// TODO: 等待设计资源替换
```

禁止：

- 使用网络占位图片。
- 使用临时 SVG 图标。
- 使用无意义的测试资源。

# 可观测性

涉及：

- AI
- 长耗时任务
- 文件上传
- 数据同步
- 批量操作

建议保留：

- 请求日志
- 错误日志
- TraceId
- TaskId

发生异常时：

- 给出明确错误提示。
- 保留失败原因。
- 提供重试能力。

---

# 第三方库使用原则

优先使用项目已有依赖。

例如：

| 场景     | 推荐       |
| -------- | ---------- |
| HTTP     | Axios      |
| 时间     | dayjs      |
| 工具函数 | lodash-es  |
| 状态管理 | Pinia      |
| 路由     | Vue Router |

禁止重复引入同类型第三方库。
