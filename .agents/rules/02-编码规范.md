# 编码规范

## 基本原则（NON-NEGOTIABLE）

所有新增代码必须遵循：

- 保持代码可读性
- 保持代码可维护性
- 保持职责单一
- 保持命名清晰
- 保持类型完整
- 优先复用已有代码
- 避免重复实现（DRY）
- 优先遵循项目已有实现方式

禁止为了完成需求而破坏已有代码结构。

---

# TypeScript

所有代码必须使用 TypeScript。

禁止：

- JavaScript
- any（无法避免时使用 unknown，并说明原因）

优先：

- interface
- type
- 泛型
- 类型推导

例如：

```ts
interface User {
  id: number;
  name: string;
}
```

---

# 命名规范

变量：

```
camelCase
```

例如：

```ts
userInfo

currentPage

searchKeyword
```

函数：

使用动词开头。

例如：

```ts
getUserList

createUser

updateUser

deleteUser

handleSubmit

handleSearch
```

组件：

```
PascalCase
```

例如：

```
UserTable.vue

SearchForm.vue
```

常量：

```
UPPER_SNAKE_CASE
```

例如：

```ts
DEFAULT_PAGE_SIZE

MAX_UPLOAD_SIZE
```

枚举：

使用有业务含义的名称。

例如：

```ts
UserStatus

OrderType
```

禁止：

```ts
data

obj

temp

test

aaa

bbb
```

---

# 函数规范

一个函数只负责一件事情。

推荐：

```ts
getUserList()

formatDate()

validateForm()
```

禁止：

一个函数同时：

- 请求接口
- 修改状态
- 操作 DOM
- 格式化数据

职责应拆分。

---

# 方法长度

推荐：

一个函数控制在：

```
30~50 行
```

超过时应考虑拆分。

---

# 组件职责

一个组件负责一个业务功能。

例如：

```
SearchForm

UserTable

CreateDialog
```

禁止：

一个组件承担：

- 查询
- 表格
- 编辑
- 删除
- 导出

全部逻辑。

公共能力应抽离。

---

# 重复代码

出现两次以上相同逻辑时：

优先考虑：

- utils
- composables
- hooks（Composable）
- 公共组件

禁止复制粘贴代码。

---

# Magic Number

禁止：

```ts
if (status === 1)
```

推荐：

```ts
const ENABLE = 1;

if (status === ENABLE)
```

或者：

```ts
enum UserStatus {
    Enable = 1
}
```

---

# 条件判断

避免：

```ts
if (...) {

} else {

}
```

嵌套过深。

推荐：

提前返回。

例如：

```ts
if (!user) {
    return;
}

doSomething();
```

---

# Optional Chaining

推荐：

```ts
user?.name
```

替代：

```ts
user && user.name
```

---

# Null 判断

推荐：

```ts
if (!data) {
    return;
}
```

避免：

```ts
if (data === null || data === undefined)
```

---

# 异步规范

统一使用：

```
async / await
```

避免：

```ts
.then()

.catch()
```

连续调用。

必须：

```ts
try {
    ...
} catch (error) {
    ...
}
```

处理异常（如项目已有统一封装，则遵循项目规范）。

---

# 导入规范

优先顺序：

1. Vue
2. 第三方库
3. 项目公共模块
4. 当前模块

例如：

```ts
import { computed } from "vue";

import { ElMessage } from "element-plus";

import dayjs from "dayjs";

import { getUserList } from "@/http/user";

import type { User } from "@/interfaces/user/model";
```

---

# 类型导入

类型必须使用：

```ts
import type
```

例如：

```ts
import type { User } from "@/interfaces/user/model";
```

禁止：

```ts
import { User } ...
```

---

# 注释规范

优先使用：

能够表达业务含义的变量名。

避免大量注释。

只有：

- 复杂算法
- 特殊业务
- 临时兼容方案

才允许增加注释。

禁止：

```ts
// 获取用户

const user = ...
```

这种无意义注释。

---

# 日志规范

开发阶段：

允许：

```ts
console.log()
```

提交前：

必须删除。

禁止：

```ts
console.log

console.table

console.time
```

保留到生产代码。

---

# AI 开发约定

AI 生成代码时必须：

- 优先复用已有代码。
- 保持与项目现有风格一致。
- 不重复封装已有能力。
- 不生成无意义代码。
- 不生成未使用变量。
- 不生成未使用 import。
- 不生成死代码。
- 不修改无关代码。
- 尽可能保持最小修改范围。
