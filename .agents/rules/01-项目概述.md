# 项目概述

## 项目定位

本项目是一个基于 **Vue 3 + TypeScript + Vite** 的单页应用（SPA）。

项目采用组件化、模块化和类型安全的开发模式，所有新增代码必须遵循统一的工程规范，保证代码风格一致、目录结构统一、易于维护。

---

# 技术栈

| 领域      | 技术                                          | 说明                                         |
| --------- | --------------------------------------------- | -------------------------------------------- |
| UI 框架   | Vue 3                                         | 强制使用 Composition API +`<script setup>` |
| 类型系统  | TypeScript 5.x                                | 强制使用，禁止 JavaScript                    |
| 构建工具  | Vite 5.x                                      | 项目构建工具                                 |
| 路由管理  | Vue Router 4.x                                | 官方路由方案                                 |
| 状态管理  | Pinia                                         | 项目统一状态管理方案                         |
| UI 组件库 | Element Plus                                  | 所有交互组件优先使用 Element Plus            |
| 图标库    | @element-plus/icons-vue                       | Element Plus 官方图标库                      |
| HTTP 请求 | Axios                                         | 所有请求统一基于项目封装的 Axios 实例        |
| 样式方案  | SCSS                                          | 推荐使用 SCSS，禁止大量全局样式              |
| 工具函数  | lodash-es                                     | 优先复用 lodash-es，避免重复实现             |
| 时间处理  | dayjs                                         | 所有日期时间统一使用 dayjs                   |
| 自动导入  | unplugin-auto-import、unplugin-vue-components | 已开启自动导入                               |

---

# 开发原则（NON-NEGOTIABLE）

所有新增代码必须遵循以下原则：

- 保持组件化开发
- 保持类型完整
- 保持目录结构统一
- 优先复用已有组件
- 优先复用已有工具函数
- 优先遵循项目已有实现方式
- 避免重复代码
- 保持高内聚、低耦合

开发前，应优先阅读同目录已有代码，保持一致的编码风格。

---

# Vue 开发规范

## Composition API（NON-NEGOTIABLE）

所有 Vue 页面和组件必须使用：

```vue
<script setup lang="ts">
```

禁止：

- Options API
- JavaScript
- Class Component

---

## TypeScript（NON-NEGOTIABLE）

所有新增代码必须使用 TypeScript。

禁止：

- .js
- .jsx

优先：

- interface
- type
- 泛型
- 类型推导

禁止滥用：

```ts
any
```

只有无法表达类型时允许：

```ts
unknown
```

---

# 组件开发

所有页面和组件必须：

- 保持单一职责
- 控制组件体积
- 公共能力优先抽离
- 保持 Props 类型完整
- 保持 Emits 类型完整

组件命名采用：

```
PascalCase
```

例如：

```
UserTable.vue

SearchForm.vue

CreateDialog.vue
```

---

# Element Plus

项目统一使用：

```
Element Plus
```

要求：

- 优先使用官方组件
- 优先使用官方图标
- 保持 Element Plus 默认交互方式
- 禁止重复封装已有组件能力

如果项目已有二次封装组件：优先使用项目组件。

---

# Auto Import

项目已开启：

- unplugin-auto-import
- unplugin-vue-components

对于自动导入的 API：

禁止重复编写：

```ts
import {
    ref,
    reactive,
    computed,
    watch,
    onMounted,
    onUnmounted
} from "vue";
```

对于自动导入的 Element Plus API：

禁止重复 import。

应遵循项目 Auto Import 配置。

---

# Pinia

所有全局共享状态统一使用：

```
Pinia
```

禁止：

- Vuex
- EventBus
- window 全局变量

Store 应保持：

- 单一职责
- 模块化
- 状态最小化

禁止在 Store 中编写大量业务逻辑。

---

# Vue Router

统一使用：

```
Vue Router
```

页面必须通过路由访问。

禁止：

- 多套路由方案
- 手动维护页面切换

路由按业务模块组织。

---

# HTTP 请求

所有 HTTP 请求统一使用：

```
Axios
```

统一使用项目封装后的 Axios 实例。

例如：

```ts
import request from "@/utils/request";
```

禁止：

```ts
import axios from "axios";

axios.get(...)

axios.post(...)
```

所有 HTTP 请求统一放置于：

```
src/http
```

禁止：

- 页面直接请求
- Component 直接请求
- Pinia Store 直接请求

涉及接口开发时，应遵循：

```
.agents/rules/05-API规范.md
```

---

# 样式规范

统一使用：

```
SCSS
```

推荐：

- scoped
- BEM 命名
- CSS Variables（如项目已使用）

禁止：

- 大量全局样式
- 行内 style
- !important 滥用

公共样式应统一抽离。

---

# 工具库

优先复用：

- lodash-es
- dayjs

禁止重复实现已有能力。

例如：

数组处理：

```
lodash-es
```

日期处理：

```
dayjs
```

---

# AI 开发约定

本项目采用 AI 辅助开发。

涉及以下场景时，应优先使用对应能力：

| 场景     | 推荐方式            |
| -------- | ------------------- |
| 新增需求 | OpenSpec            |
| UI 实现  | Figma MCP           |
| 接口开发 | Apifox MCP          |
| 页面调试 | Chrome DevTools MCP |

AI 生成代码时必须：

- 优先遵循 `.agents/rules` 中的所有规则。
- 保持与项目现有代码风格一致。
- 优先复用已有组件、工具函数和业务模型。
- 禁止生成与项目规范冲突的代码。
